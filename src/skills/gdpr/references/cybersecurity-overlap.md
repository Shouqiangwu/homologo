# 与R155网络安全的交叉

> GDPR ↔ UNECE R155 CSMS

---

## 交叉关系概述

R155网络安全和GDPR数据保护有重叠但不等同：

| 维度 | R155 网络安全 | GDPR 数据保护 |
|------|-------------|-------------|
| 保护对象 | 车辆系统和通信的安全性 | 个人数据的合法处理和权利保护 |
| 认证层级 | 组织级CSMS证书 + 车型级型式认证 | 无认证要求，但需DPIA、DPO、记录等 |
| 数据保护角度 | CSMS中包含"数据保护"作为安全目标之一 | 全面的数据主体权利和控制者义务 |
| 违规后果 | 无法获得型式认证 → 不能在EU销售 | 罚款最高€20M或4%全球营收 |

## CSMS中的数据保护要求

R155 Annex 5的威胁类别中，以下与GDPR直接相关：

| R155威胁类别 | GDPR对应要求 |
|-------------|-------------|
| 未授权访问车辆数据 | GDPR Art.32 — 适当的安全措施 |
| 数据泄露 | GDPR Art.33-34 — 数据泄露通知义务（72小时内通知DPA） |
| 通信信道数据劫持 | GDPR Art.32 — 传输加密 |
| 第三方未授权访问 | GDPR Art.28 — 处理者合同要求 |

## 协同合规建议

| 建议 | 说明 |
|------|------|
| **统一TARA和DPIA** | R155要求的TARA（威胁分析和风险评估）和GDPR要求的DPIA可合并进行 |
| **安全措施共享** | R155的安全措施（加密、访问控制等）同时满足GDPR Art.32 |
| **事件响应整合** | R155网络安全事件响应和GDPR数据泄露通知可整合为统一流程 |
| **供应商管理** | R155供应链安全管理和GDPR处理者合同可同步执行 |

## DPO（数据保护官）要求

| 场景 | 是否需要DPO | 依据 |
|------|-----------|------|
| 中国车企在EU设有实体（子公司/分公司） | 可能需要 | GDPR Art.37 — 大规模处理、核心业务涉及系统性监控 |
| 中国车企无EU实体但在EU销售联网车辆 | 需要EU代表（Art.27） + 可能需要DPO | 通过远程向EU数据主体提供服务 |

## 合规检查清单

- [ ] R155 CSMS中是否包含数据保护目标？
- [ ] TARA和DPIA是否协调进行？
- [ ] 网络安全事件响应流程是否包含GDPR数据泄露通知（72小时）？
- [ ] 是否在EU任命了DPO或EU代表？
