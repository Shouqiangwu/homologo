# IDIS平台注册与维护

> IDIS Consortium规则 + ELV Art.8执行要求

---

## IDIS组织架构

```
IDIS Consortium（瑞士法人）
├─ 会员OEM（Data Providers）
│   ├─ 欧洲OEM（VW, Stellantis, BMW, Daimler, Volvo...)
│   ├─ 日韩OEM（Toyota, Honda, Nissan, Hyundai, Kia...)
│   ├─ 美国OEM（Ford, GM...)
│   └─ 中国OEM（逐步加入：MG/SAIC, BYD, Geely, NIO...)
│
├─ 技术委员会（Technical Committee）
│   └─ 维护数据schema、审核标准
│
├─ 数据运营（Data Operations）
│   └─ 日常平台管理、数据审核
│
└─ 使用方（Data Users）
    ├─ ATF（授权处理设施）
    ├─ 专业拆解企业
    ├─ 第一响应者（消防/救援——部分数据）
    └─ 监管机构
```

## OEM注册流程

### Step 1：初步评估

| 评估项 | 说明 |
|------|------|
| 是否受ELV Art.8约束？ | 所有M1/N1 OEM |
| 年产量 | 小批量OEM（<1,000辆）可能有豁免，但强烈建议仍注册 |
| EU市场战略 | 如未来5年计划进入EU，提前注册 |
| 预算 | 年订阅费+数据准备费约€20,000-€100,000/年 |

### Step 2：申请入会

```
访问 www.idis2.com → "Become a Member"
    ↓
填写申请表：
├─ OEM公司信息
├─ 车型目录
├─ 预计数据提交时间
└─ 联系人信息
    ↓
IDIS Consortium审核（通常2-4周）
    ↓
签署Member Agreement（含数据标准、IP条款、费用）
    ↓
支付首年订阅费
    ↓
获得数据提交权限
```

### Step 3：技术配置

| 配置 | 说明 |
|------|------|
| User账号 | 管理员 + 数据提交员 + 审核员 |
| API访问 | 可选，用于自动化数据上传 |
| 证书配置 | HTTPS证书，用于安全上传 |
| 本地工具 | IDIS Data Provider Tool（桌面软件） |

### Step 4：首次数据提交

```
OEM内部组织数据编写
    ↓
使用IDIS Data Provider Tool录入
    ↓
内部QA
    ↓
上传到IDIS portal
    ↓
IDIS审核员检查（1-2周）
    ↓
修正反馈
    ↓
发布给ATF可见
```

## 订阅费结构

### 年费（OEM Data Provider）

| OEM规模 | 年订阅费（估算） |
|---------|----------------|
| 小型（<10,000辆EU销量） | €5,000-15,000 |
| 中型（10k-100k） | €15,000-40,000 |
| 大型（>100k） | €40,000-100,000+ |

**包含：** 无限数据提交、审核服务、平台更新。

**不含：** 数据准备成本（OEM内部工作）、翻译、特殊附件（视频等）。

### 数据使用方费用（ATF）

| ATF规模 | 年订阅 |
|---------|-------|
| 小型（<500辆/年） | €500-1,500 |
| 中型（500-5,000辆/年） | €1,500-5,000 |
| 大型（>5,000辆/年） | €5,000-15,000 |

**理念：** OEM和使用方双向付费，支撑平台运营。

| 功能 | 说明 |

| 图像嵌入 | 拆解步骤配图 |

用于大规模数据同步：

## 数据维护SLA

### 首次提交时限

| 阶段 | 时限 |
|------|------|
| 车型型式认证完成 | T0 |
| 车型EU上市 | T0 + 0-6月 |
| IDIS初始数据提交 | T0 + 6月（必须） |
| IDIS数据发布 | T0 + 6-12月（审核完成） |

### 更新时限

| 触发 | 更新时限 |
|------|---------|
| 车型改款 | 改款上市后3-6月 |
| 软件版本重大变更 | 3-6月 |
| 法规变化（如Annex II修订） | 法规生效后3-6月 |
| ATF反馈问题 | 根据严重性，1-3月 |
| 安全警告（如火灾风险） | 立即 |

### 停产后维护

| 情况 | 要求 |
|------|------|
| 停产但仍有大量在用车 | 持续维护≥15年 |
| 停产且在用车极少 | 可申请"legacy status"减少更新 |
| OEM破产 | IDIS保留历史数据，不再更新 |

## 数据审核标准

### 自动校验（上传时）

- 必填字段完整性
- 数据格式正确性
- 图像质量（分辨率、格式）
- 文件大小限制

### 人工审核（IDIS团队）

- 数据准确性（与公开资料对比）
- 安全警告完整性
- 专业术语正确性
- 图示清晰度
- 与同类车型的一致性

### ATF反馈

ATF在实际使用中发现问题可提交反馈：
- 步骤错误
- 缺失信息
- 图示不清
- 工具要求不正确

IDIS转发OEM，OEM需修正。

## 知识产权和数据共享

### 数据所有权

- OEM保留数据所有权
- IDIS Consortium获得使用权（仅用于ELV拆解目的）
- ATF仅可在拆解活动中使用数据

### 限制

- 不可用于竞争分析
- 不可公开发布
- 不可再分发

## 合规检查清单

- [ ] OEM已注册IDIS Member？
- [ ] Member Agreement已签署？
- [ ] 年订阅费已缴纳？
- [ ] 本地工具已配置？
- [ ] 首次数据上传在上市后6月内？
- [ ] 所有EU市场车型都在IDIS？
- [ ] 改款/软件变更的更新及时？
- [ ] 停产车型数据仍维护？
- [ ] ATF反馈有响应机制？
- [ ] 内部有专人负责IDIS数据？

## 常见不合规情况

1. **小OEM未注册**：年EU销量<5,000辆的OEM觉得"不值得"付订阅费，导致违反Art.8。
2. **注册晚**：EU上市后才开始注册流程，数据到位滞后1-2年。
3. **仅注册不提交**：有账号但实际数据准备拖延。
4. **停订阅**：年度订阅费未续，数据下线。
5. **兼职维护**：无专职IDIS负责人，信息老旧。
6. **ATF反馈忽视**：收到ATF反馈但无响应流程。
7. **多车型合并**：不同车型用同一份数据，掩盖差异。
