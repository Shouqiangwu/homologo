# OTA更新影响评估

> UNECE R156（SUMS） + EU 2018/858 + GSR实施指引

---

## OTA更新的合规框架

OTA（Over-The-Air）更新作为现代车辆的必备能力，受以下法规共同约束：

```
UNECE R156 (SUMS)
├─ 要求OEM建立SUMS（Software Update Management System）
├─ 型式认证时需有SUMS证书
└─ 每次OTA需按R156分类管理

EU 2018/858 Art.34/37
├─ 型式认证变更管理
├─ 实质性变更需重新认证
└─ 非实质变更可走简化程序

GSR Implementing Acts
├─ ISA软件规则（EU 2021/1958）
├─ DDAW规则（EU 2021/1341）
└─ 各ADAS系统的特定软件要求
```

## OTA更新的分类（R156 + 行业实践）

### Category A - 安全关键，需重新型式认证

| 变更类型 | 示例 |
|---------|------|
| AEB标定曲线变更 | 触发条件、制动压力 |
| ISA算法升级 | 限速识别逻辑 |
| ELKS转向策略变更 | 力矩曲线 |
| EDR记录项修改 | 数据字段或触发事件 |
| 安全气囊部署逻辑 | 触发条件、时序 |
| 高压系统控制 | 电池管理、充电策略的安全相关 |

**流程：** 认证机构预审 → 技术文件更新 → 可能重新测试 → 重新颁发TA证书

### Category B - 安全相关，需通知认证机构

| 变更类型 | 示例 |
|---------|------|
| ADAS参数微调（在认证范围内） | 摄像头曝光、雷达灵敏度 |
| HMI警告显示优化 | 颜色、时长 |
| 故障诊断逻辑 | 新增DTC代码 |
| 错误处理改进 | 降级模式 |
| 电池充电曲线优化 | SOC管理 |

**流程：** OEM自评估 → 通知认证机构 → 更新RXSWIN → 归档

### Category C - 非安全相关，仅记录

| 变更类型 | 示例 |
|---------|------|
| 信息娱乐系统 | UI、媒体、导航 |
| 蓝牙/WiFi功能 | 连接性 |
| 座椅按摩、氛围灯 | 舒适功能 |
| 多语言支持 | 仅显示 |

**流程：** OEM内部记录 → 无需通知 → 用户正常推送

## RXSWIN体系

### RXSWIN（Regulation X Software Identification Number）

每个受UNECE法规约束的软件模块分配一个唯一RXSWIN：

| 法规 | RXSWIN示例 |
|------|----------|
| R155 | RXSWIN-R155-2024-0142 |
| R156 | RXSWIN-R156-2024-0143 |
| R157 ALKS | RXSWIN-R157-... |
| R152 AEB | RXSWIN-R152-... |
| R130 ELKS | RXSWIN-R130-... |

### RXSWIN管理

```
某车型型式认证时登记初始RXSWIN
        ↓
Category A更新 → RXSWIN变更 → 重新认证
Category B更新 → RXSWIN变更 → 通知
Category C更新 → RXSWIN不变 → 无动作
        ↓
车辆VIN-RXSWIN映射维护
        ↓
ISC测试时验证RXSWIN与实际软件一致
```

## SUMS（Software Update Management System）

### SUMS的要素

按R156 Annex 3：

| 要素 | 要求 |
|------|------|
| **SUMS证书** | OEM的SUMS经认证机构审计合格 |
| **过程文档** | 软件开发、测试、部署的SOP |
| **变更管理** | 每次更新的分类、影响分析、决策 |
| **证据记录** | 测试报告、认证机构通知 |
| **目标车辆识别** | 确保更新推送到正确车辆 |
| **回退能力** | 更新失败时恢复 |
| **用户通知** | 更新前告知用户 |
| **网络安全** | 签名验证、加密传输（R155协调） |
| **成功验证** | 确认更新成功部署 |

### SUMS审计

| 阶段 | 审计方 |
|------|--------|
| 初次认证 | Technical Service + TAA |
| 周期复审 | 每3年 |
| 变更触发 | 重大流程变更 |
| 事件触发 | 重大OTA事件后 |

## 影响评估流程

### Step 1：变更识别

```
开发团队提出软件变更 → 
    ├─ 自动扫描：变更影响了哪些ECU/功能？
    ├─ 人工评估：是否涉及R155/R156/R152等法规？
    └─ 初步分类：A/B/C
```

### Step 2：技术评估

| 评估维度 | 问题 |
|---------|------|
| 功能变化 | 是否改变了型式认证时的行为？ |
| 性能变化 | 是否影响关键性能（制动距离、识别率）？ |
| 安全影响 | 是否降低了任何安全裕度？ |
| 网络安全 | 是否引入新攻击面？ |
| 其他法规 | 是否影响排放、GDPR、EDR等？ |

### Step 3：合规决策

| 决策 | 触发 |
|------|------|
| 走Category A重新认证 | 任一核心评估显示实质性变化 |
| 走Category B通知 | 安全相关但参数在原范围内 |
| 走Category C常规 | 确认无安全/合规影响 |

### Step 4：认证机构互动

| 场景 | 与认证机构的沟通 |
|------|----------------|
| Category A | 预审会议 → 提交技术包 → 补充测试 → 重新颁发 |
| Category B | 正式通知（通常邮件/portal）→ 认证机构确认或提问 |
| Category C | 归档备查 |

### Step 5：部署和监控

```
OTA推送 → 用户接受 → 更新安装 → 验证成功
        ↓
     回收失败数据 → 必要时触发回滚
        ↓
     更新VIN-RXSWIN映射
        ↓
     如有ISC数据变化，同步ISC团队
```

## 常见OTA场景的影响评估

### 场景1：AEB性能优化

OEM发现AEB在特定条件（湿路+夜间）下制动距离偏长，希望通过OTA优化：

| 分析 | 结论 |
|------|------|
| 是否改变型式认证功能？ | 是（性能边界变化） |
| 分类 | **Category A** |
| 流程 | 重新做R152关键场景测试 → 认证机构批准 → 推送 |

### 场景2：ISA地图数据包更新

OEM推送新版地图数据（限速标识数据库）：

| 分析 | 结论 |
|------|------|
| 是否改变识别算法？ | 否（仅数据更新） |
| 分类 | **Category C** |
| 流程 | 内部记录 → 推送 |

### 场景3：EV电池充电曲线优化

OEM通过OTA调整BMS充电曲线，提高充电速度：

| 分析 | 结论 |
|------|------|
| 是否影响安全（热失控风险）？ | 可能 |
| 是否影响续航/能耗申报？ | 是（WLTP值可能需重测） |
| 分类 | **Category A**（需充电安全和能耗重测） |

### 场景4：网络安全补丁

修复CVE-2024-XXXX漏洞：

| 分析 | 结论 |
|------|------|
| 是否涉及R155 CSMS？ | 是 |
| 紧急度 | 高（漏洞利用风险） |
| 分类 | **Category A**（R155相关） |
| 流程 | R155 CSMS加急通报 + 重新RXSWIN |

## 对ISC的影响

### OTA后的ISC重新评估

| 情境 | 重新评估要求 |
|------|------------|
| Category A更新 | 所有已部署车辆进入新RXSWIN → ISC测试需覆盖新版本 |
| Category B更新 | 更新后的性能数据纳入ISC监控基线 |
| Category C更新 | 无ISC影响 |

### ISC样本选择

OTA更新后3-6个月内，ISC样本应优先覆盖：
- 已接收新版本的车辆
- 尚未接收新版本的车辆（比较基线）

## 合规检查清单

- [ ] 是否有SUMS证书？
- [ ] SUMS是否在过去3年内被审计？
- [ ] 每次OTA是否有变更分类流程？
- [ ] 是否有自动化工具识别法规影响？
- [ ] Category A更新是否触发重新认证流程？
- [ ] Category B更新是否通知认证机构？
- [ ] RXSWIN是否在每次安全相关更新时更新？
- [ ] 车辆VIN-RXSWIN映射是否可查？
- [ ] 回退能力是否测试过？
- [ ] 网络安全（R155）和软件更新（R156）协调是否到位？
- [ ] OTA后ISC监控基线是否更新？
- [ ] 用户通知机制是否满足R156要求？

## 常见不合规情况

1. **全部OTA按Category C处理**：任何安全相关变化都走"简化"流程，认证机构抽查时发现多个Category A未通报。
2. **RXSWIN管理混乱**：不同ECU有不同版本号体系，无统一RXSWIN映射。
3. **紧急网络安全补丁未走R155流程**：为求快速修复漏洞，跳过R155通报。
4. **回退失败**：OTA失败车辆处于中间状态，既不是原版也不是新版。
5. **部分车辆未推送成功**：长期未联网车辆处于老版本，RXSWIN映射与实际不符。
6. **第三方软件组件变更未跟踪**：地图数据、语音识别等第三方组件更新未纳入SUMS。
7. **OTA影响其他法规忽略**：BMS更新影响电池寿命（Battery Reg）未被考虑。
