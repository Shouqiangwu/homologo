# GSR 框架概述与实施时间表

> EU 2019/2144 + Annex II

---

## 法规结构

EU 2019/2144是**框架法规**，本身不包含具体技术要求，而是：
1. 定义了新车必须配备的安全系统清单（Art.4-11）
2. 授权欧委会通过委托法规（Delegated Acts）和实施法规（Implementing Acts）制定具体技术要求
3. 引用UNECE法规作为技术标准来源
4. 在Annex II中设定所有要求的强制实施日期

**关键条款索引：**
- Art.4：一般要求（适用所有M/N/O类）
- Art.5：M1/N1特殊要求（AEB、ELKS、扩大头部碰撞保护区）
- Art.6：ISA技术要求
- Art.7：DDAW和ADDW技术要求
- Art.8：EDR技术要求
- Art.9：TPMS（轮胎压力监测）
- Art.10：网络安全和软件更新
- Art.11：完全自动驾驶车辆
- Art.16：实施日期（引用Annex II）
- Annex I：UNECE法规与车辆类别的对应关系
- Annex II：所有要求的实施时间表（核心！）

## 中国车企常见合规差距

以下是中国M1类BEV车企进入EU市场时最常见的GSR合规差距：

1. **ISA限速标志数据库**：系统仅支持中国GB5768标志，未覆盖EU/EEA成员国限速标志（维也纳公约体系）。需要为每个EU成员国适配标志识别数据库。ISA准确率要求≥90%。

2. **AEB行人/骑行者检测**：R152.01/02扩展了行人和骑行者检测要求，部分中国车企的AEB系统在Euro NCAP评级中表现良好但可能未针对R152的特定测试场景（如横穿行人5km/h、横穿骑行者15km/h）做合规标定。

3. **R155网络安全**：需要通过CSMS（网络安全管理系统）审核，这是组织层面的认证，不是单个车型的测试。中国车企需在EU建立或委托网络安全管理体系。

4. **R156软件更新**：OTA更新必须通过SUMS（软件更新管理系统）审核，确保更新不影响型式认证合规性。中国车企习惯的高频OTA模式需适配EU监管框架。

5. **EDR数据隐私**：EDR不得记录VIN后4位（EU 2019/2144 Art.8(5)），不得使数据可与特定车辆关联。部分中国车企的EDR方案需修改数据记录策略。

6. **ELKS与中国法规差异**：R130的ELKS（紧急车道保持，主动纠正车辆回车道内）与中国法规中的LDW（仅警告）有本质区别，系统功能需升级。

## 小批量车辆和特殊用途车辆

EU 2022/2236规定了小批量（Small Series）和特殊用途（Special Purpose）车辆的GSR要求：
- 小批量定义：M1/N1类，年产≤1,500台
- 适用范围：部分GSR要求可豁免或延期
- 新车型日期：2024-07-07
- 所有新车日期：2026-07-07
- 中国车企注意：仅当EU年注册量≤1,500台且车型非大批量车型的衍生型时可申请
