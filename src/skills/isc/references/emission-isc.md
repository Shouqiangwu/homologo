# 排放ISC（在用车排放符合性）

> EU 2018/1832 + EU 2017/1151 Annex II + EU 2018/858 Art.52

---

## 排放ISC的法规体系

```
EU 2018/858（框架法规）
├─ Art.52：市场监督的一般原则
├─ Art.53：OEM的义务
└─ Art.54：不合规后的措施

EU 2018/1832（排放ISC具体实施）
├─ WLTP排放ISC程序
├─ RDE（Real Driving Emissions）ISC程序
└─ CO2/油耗ISC监控

EU 2021/392（OBFCM数据报告）
└─ 实际CO2/能耗的VIN级数据
```

## 排放ISC的测试对象

### 排放项目

| 项目 | 测试方法 | 关键阈值 |
|------|---------|---------|
| **CO2** | WLTP | 偏离型式认证值需在conformity factor内 |
| **NOx** | WLTP + RDE | Euro 6d限值 × CF |
| **PM/PN** | WLTP + RDE | Euro 6d限值 × CF |
| **CO** | WLTP | Euro 6限值 × CF |
| **THC/NMHC** | WLTP | Euro 6限值 × CF |
| **NH3** | WLTP | 新纳入Euro 7（2026后） |

### Conformity Factor（CF）

RDE测试中允许的实际值与型式认证值的偏离：

| 项目 | Euro 6d CF | Euro 7 CF（2026-） |
|------|-----------|-------------------|
| NOx | 1.43（最终） | 待定，可能更严 |
| PN | 1.5 | 待定 |
| CO/THC | 不限（WLTP限值内） | 新增RDE CF |

## 测试样本选择

### 选车规则

| 维度 | 要求 |
|------|------|
| 车龄 | 15,000 km - 100,000 km（正常使用阶段）|
| 数量 | 每个车型家族（family）至少3辆 |
| 代表性 | 覆盖不同地理/使用条件 |
| 来源 | OEM提供名单，监管或OEM抽取 |

### Family定义

按EU 2017/1151 Annex XXI定义家族：
- 引擎相同
- 变速箱类型相同
- 后处理系统相同
- 整车质量相近（±10%）
- 气动特性相近

同一家族内ISC测试结果可相互代表。

## 测试程序详细

### WLTP ISC（实验室）

```
Step 1: 车辆接收 → 检查使用历史（保养记录、里程、软件版本）
Step 2: 预处理 → 按正常使用状态（不允许特别调教）
Step 3: WLTP Chassis Dyno测试（按型式认证程序）
Step 4: 各污染物测量
Step 5: 比较型式认证值
Step 6: 若超标：
   ├─ 单车超标 → 记录，需更多车辆确认
   ├─ 家族统计超标 → 触发调查
   └─ 系统性超标 → 可能强制召回
```

### RDE ISC（实际道路）

```
Step 1: 在公共道路上行驶（路况符合RDE窗口）
Step 2: PEMS（便携式排放测量系统）记录
Step 3: 测试时长：90-120分钟
Step 4: 覆盖城市（34%）+ 乡村（33%）+ 高速（33%）
Step 5: 按RDE算法计算排放窗口
Step 6: 对比CF限值
```

## OEM的ISC义务

### 年度ISC监控计划

OEM需提交：

| 内容 | 详情 |
|------|------|
| 车型覆盖 | 哪些车型/家族将在当年ISC测试 |
| 测试数量 | 按销量比例（通常每10,000辆1-5辆） |
| 测试地点 | EU Technical Service或OEM授权实验室 |
| 时间表 | 季度计划 |

### 测试结果报告

- 每季度向认证机构（TAA）提交测试结果
- 超标需24小时内通知
- 年度汇总报告

### 不合规响应

```
单车超标 → 记录+调查因果
    ↓
3车/家族超标 → 扩大抽样
    ↓
系统性超标确认 → 
    ├─ OEM主动纠正（软件更新、召回）
    ├─ 认证机构强制纠正
    └─ 严重时：撤销认证 + 禁止新车销售
```

## OBFCM（On-Board Fuel Consumption Monitoring）

### 法规要求

EU 2021/392起所有新M1/N1必须配备OBFCM：

| 数据项 | 说明 |
|------|------|
| 累计燃油消耗 | 自出厂总燃油（L或kWh） |
| 累计电能消耗 | BEV/PHEV的累计电耗 |
| 累计里程 | 总里程数 |
| 累计CO2 | 计算值 |

### 数据读取

- 年度车检（TÜV/DEKRA）时通过OBD读取
- 成员国汇总上报EEA
- EEA比对型式认证CO2与实际CO2

### 阈值

| 偏差 | 响应 |
|------|------|
| WLTP+20%以内 | 正常 |
| +20-30% | 监控，调查是否PHEV用户行为问题 |
| >+30% | 触发OEM调查 |
| >+50% | 可能强制软件更新或召回 |

**PHEV特别关注：** PHEV的WLTP CO2依赖UF，若实际使用中燃油模式占比远高于UF假设，OBFCM数据会显示大幅偏离。EU 2023/443修订了PHEV UF算法，2025起使用OBFCM校准的UF。

## 市场监督（Market Surveillance）

### 监管机构的权力

- 抽查销售中或使用中的车辆
- 要求OEM提供ISC数据
- 独立测试（无需OEM配合）
- 发布超标调查结果

### 著名案例

| 年份 | 事件 | 结果 |
|------|------|------|
| 2015 | VW柴油门（Dieselgate） | 400亿美元罚款+召回1100万辆 |
| 2018 | Daimler/Fiat排放争议 | 召回+罚款 |
| 2020-2023 | 多家OEM的PHEV实际排放调查 | UF算法修订 |

## 与其他ISC的关系

| ISC类型 | 基础法规 | 负责领域 |
|---------|---------|---------|
| **排放ISC** | EU 2018/1832 | 污染物、CO2 |
| **ADAS ISC** | EU 2019/2144 + GSR | AEB/ISA/ELKS等 |
| **R156 SUMS ISC** | UNECE R156 | OTA更新影响 |
| **电池ISC** | Battery Reg Art.10 | 电池性能、寿命 |
| **噪声ISC** | UNECE R51 | 车内外噪声 |

## 合规检查清单

- [ ] 是否建立了年度排放ISC监控计划？
- [ ] 每个车型家族是否有≥3辆的样本？
- [ ] WLTP和RDE测试是否都覆盖？
- [ ] 样本选择是否覆盖15,000-100,000 km使用阶段？
- [ ] OBFCM数据采集是否可靠？
- [ ] 超标后的响应流程是否建立？
- [ ] 是否有与认证机构（TAA）的定期沟通？
- [ ] 季度报告是否按时提交？
- [ ] PHEV的UF和OBFCM数据是否监控？
- [ ] 柴油车的NOx DeNOx系统在ISC中是否特别关注？

## 常见不合规情况

1. **PHEV实际CO2大幅偏离**：WLTP宣称CO2 30 g/km但OBFCM显示实际100+ g/km——用户长期不充电使用。
2. **OBFCM数据读取链路断裂**：中国出口车辆的OBD协议与EU年检设备不兼容。
3. **DPF再生干扰RDE**：柴油车DPF主动再生期间排放远超CF，测试时机选择不当。
4. **尿素液（AdBlue）不足**：SCR系统因用户未补充尿素而失效，导致NOx超标。
5. **老车型软件版本滞后**：新软件修复了排放问题但未强制推送，老软件车型ISC失败。
6. **Family定义过宽**：不同配置的变体超出真正的相似性，小样本结果无法代表全家族。
7. **样本车非"典型使用"**：选取保养过度/里程过低的车辆，未反映真实使用。
8. **RDE测试窗口操纵**：选择低温、无坡道、平稳驾驶以规避严苛条件。
