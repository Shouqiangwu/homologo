# EDR 事件数据记录器要求

> EU 2022/545（M1/N1）+ UNECE R169 + C(2024)5214（M2/M3/N2/N3）

---

## 适用范围和时间表

| 适用类别 | 法规依据 | 新车型强制 | 所有新车强制 |
|---------|---------|-----------|-------------|
| M1/N1 | EU 2022/545 | 2022-07-06 | 2024-07-07 |
| M2/M3/N2/N3 | R169 + C(2024)5214 | 2026-01-07 | 2029-01-07 |

## 系统定义

EDR记录碰撞前后短时间窗口内的关键车辆数据，用于事故分析和安全研究。**不是行车记录仪**（dashcam），不记录视频/音频。

## M1/N1 EDR技术要求 — EU 2022/545

### 触发条件

| 触发类型 | 条件 |
|---------|------|
| 主触发 | 气囊展开（任何气囊） |
| 可选触发 | 其他碰撞检测事件（由OEM定义） |

### 必须记录的数据元素

| 数据类别 | 具体参数 | 时间窗口 |
|---------|---------|---------|
| 车辆速度 | 纵向速度 | 碰撞前5秒至碰撞后0.3秒 |
| 制动状态 | 制动踏板位置/制动系统激活 | 同上 |
| 加速踏板 | 加速踏板位置 | 同上 |
| 转向角 | 方向盘角度 | 同上 |
| 安全带状态 | 驾驶员和前排乘客安全带扣合状态 | 碰撞时刻 |
| 气囊状态 | 各气囊展开状态和时间 | 碰撞时刻 |
| 纵向/横向加速度 | Delta-V（速度变化量） | 碰撞前至碰撞后150ms |

### 隐私保护要求 — EU 2019/2144 Art.8(4)-(5)

| 限制项 | 具体要求 | 法规条款 |
|--------|---------|---------|
| **VIN限制** | EDR**不得**记录VIN后4位数字 | Art.8(5) |
| **匿名化** | 记录的数据不得使人能将数据与特定个人或车辆关联 | Art.8(4) |
| **数据不可外传** | EDR数据存储在车辆本地，不得通过车辆接口自动传输 | Art.8(4) |
| **数据访问** | 仅通过物理接口（如OBD端口）读取 | Art.8(4) |
| **加密存储** | EDR数据应使用安全方式存储 | Art.8(4) |
| **数据覆盖** | 旧事件数据可被新事件覆盖 | 实施法规 |

### 数据读取工具要求

| 要求项 | 具体要求 |
|--------|---------|
| 标准化接口 | 通过标准化OBD-II端口读取 |
| 工具可用性 | 车企必须使EDR数据读取工具或协议对事故调查机构可用 |
| 数据格式 | 标准化输出格式 |

## 合规检查清单

- [ ] EDR是否安装在车辆中？
- [ ] 触发条件是否至少包含气囊展开？
- [ ] 是否记录所有必须的数据元素（速度、制动、加速、转向、安全带、气囊、Delta-V）？
- [ ] 数据记录时间窗口是否覆盖碰撞前5秒至碰撞后0.3秒？
- [ ] EDR是否**不**记录VIN后4位？
- [ ] EDR数据是否**不**自动通过车辆接口传输？
- [ ] 数据是否仅通过物理接口（OBD）读取？
- [ ] 数据是否加密存储？
- [ ] 是否提供了数据读取工具或协议的相关文档？
- [ ] 数据格式是否符合标准化要求？

## 常见不合规情况

1. **EDR记录了VIN完整信息**：中国国标EDR（GB 39732-2020）允许记录完整VIN，但EU明确禁止记录VIN后4位。需修改EDR软件。
2. **EDR数据通过T-BOX自动上传云端**：部分中国车企的EDR数据实时上传到OEM数据中心，违反Art.8(4)数据不可外传的要求。
3. **数据读取工具未提供**：车企未向EU事故调查机构提供EDR数据读取工具或协议。
4. **时间窗口不足**：部分系统仅记录碰撞前2-3秒数据，不满足5秒要求。
5. **Delta-V分辨率不足**：记录的速度变化量精度不满足要求。
