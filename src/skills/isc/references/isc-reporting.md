# ISC报告与记录管理

> EU 2018/858 Art.52-54 + EU 2018/1832 Art.7 + R156 Annex 3

---

## ISC报告的类型

OEM面向EU监管体系的ISC相关报告：

```
年度ISC监控计划
    ├─ 季度测试结果报告
    ├─ 即时不合规通知
    ├─ OTA相关报告
    └─ 年度综合ISC报告
```

## 年度ISC监控计划

### 提交时机

- 每年末提交次年ISC计划
- 每个认证车型家族单独编写
- 提交给颁发原始型式认证的认证机构（TAA）

### 内容要求

| 章节 | 内容 |
|------|------|
| 1. 范围 | 覆盖的车型、销售地区、注册量 |
| 2. 抽样策略 | 每车型家族样本数、选车标准 |
| 3. 测试项目 | WLTP / RDE / CO2 / ADAS / OBFCM各项 |
| 4. 测试机构 | Technical Service名单 |
| 5. 时间表 | 季度测试时间 |
| 6. 风险评估 | 已知合规风险点 |
| 7. 纠正措施预案 | 不合规情况的响应流程 |

### 典型规模

| OEM年注册量 | ISC测试车辆数 |
|-----------|-------------|
| <10,000 | 3-10辆（最小样本） |
| 10,000-100,000 | 20-50辆 |
| 100,000-500,000 | 50-200辆 |
| >500,000 | 200-1,000辆 |

## 季度测试结果报告

### 提交频次

- 每季度末后30天内提交
- 4份/年，每份覆盖前一季度

### 内容

| 部分 | 细节 |
|------|------|
| 测试执行情况 | 按计划完成百分比 |
| 每辆车测试结果 | VIN、里程、测试项、结果 |
| 合规统计 | Pass/Fail比例 |
| 趋势分析 | 与上季度对比 |
| 异常说明 | 单车异常的归因 |
| 下季度调整 | 如需扩大抽样等 |

## 即时不合规通知

### 触发条件

| 场景 | 时效 |
|------|------|
| 单车严重超标（如NOx >2x型式认证值） | 24小时内 |
| 系统性问题（3辆同家族超标） | 72小时内 |
| 安全相关ADAS失效 | 即时（通常4小时内） |
| 网络安全事件 | R155规定的时间 |

### 通知内容

- 问题描述（测试数据）
- 初步归因分析
- 影响范围（VIN清单或车型家族）
- 拟采取的纠正措施
- 时间表

### 监管响应

```
OEM通知 → TAA评估
    ↓
├─ 自主纠正可控 → 监控OEM执行
├─ 风险较高 → 要求扩大测试
└─ 严重 → 暂停销售 + 强制召回
```

## OTA相关报告

### Category A更新

| 时间节点 | 报告内容 |
|---------|---------|
| 预审阶段（更新前） | 变更说明、影响分析、测试计划 |
| 部署阶段 | 推送计划、目标车辆清单 |
| 部署后30天 | 推送成功率、问题清单 |
| 部署后6月 | ISC测试结果、性能验证 |

### Category B更新

| 时间节点 | 报告内容 |
|---------|---------|
| 部署前 | 简要通知（邮件/portal） |
| 部署后30天 | 成功率 |
| 年度 | 合并到年度ISC报告 |

## 年度综合ISC报告

### 提交时间

每年4月底前提交前一日历年的综合报告。

### 核心章节

| 章节 | 内容 |
|------|------|
| Executive Summary | 1-2页高管摘要 |
| ISC Activity | 全年测试活动统计 |
| Compliance Results | 分车型、分项的合规率 |
| Trends Analysis | 多年趋势对比 |
| OTA Activities | 全年OTA更新汇总 |
| Non-Conformities | 不合规事件清单和处理 |
| Recalls | 召回事件清单 |
| Customer Feedback | 客户投诉和保修数据的ISC相关分析 |
| Forward Look | 下年ISC重点 |

### 报告读者

- TAA（Type Approval Authority）
- 欧委会 DG GROW（市场监督）
- 市场监督机构（成员国）
- 部分信息向公众公开（如EEA排放数据）

## 记录保存要求

### 法规要求

| 记录类型 | 保存期限 |
|---------|---------|
| ISC测试原始数据 | 10年 |
| 软件版本和RXSWIN记录 | 车型停产后+10年 |
| OTA决策文档 | 10年 |
| 客户投诉记录 | 10年 |
| 召回相关文档 | 15年 |
| OEM内部ISC流程文档 | SUMS审计周期（3年+） |

### 数据格式要求

- 机器可读（避免纸质化石）
- 可检索（按VIN、时间、车型）
- 防篡改（审计轨迹）
- 长期可读（避免过时格式）

## ISC与召回的关系

### ISC触发召回的路径

```
ISC发现问题
    ↓
OEM分析：
├─ 可软件修复 → OTA推送（Category A）→ 可能避免传统召回
├─ 需硬件更换 → 强制召回
└─ 仅特定批次 → 限定召回
    ↓
召回决策通知TAA
    ↓
召回执行（通常6-18个月完成率目标90%+）
    ↓
召回效果评估
    ↓
若召回失败 → 可能强制停售 / 认证撤销
```

### 召回类型

| 类型 | 触发 | 主动性 |
|------|------|-------|
| Voluntary Safety Recall | OEM发现问题 | 主动 |
| Voluntary Non-Safety Recall | OEM改进（非强制） | 主动 |
| Mandatory Recall | 监管强制 | 被动 |
| Service Campaign | 小范围、非安全 | 主动 |

## 对中国车企的实操要点

### 报告基础设施

| 能力 | 要求 |
|------|------|
| ISC数据系统 | 整合各地Technical Service的测试结果 |
| EU法人接口 | 由EU代表正式向TAA提交报告 |
| 语言 | 报告需英文版本 |
| 时区 | 紧急通知按CET时区24h响应 |
| 格式 | 欧洲标准模板（非中国GB格式） |

### 常见组织架构

```
中国总部
├─ 法规合规部（制定策略）
├─ 测试部（安排Technical Service）
└─ 数据部（ISC数据库）
    ↕
EU法人/代表
├─ 合规对接（与TAA沟通）
├─ 报告提交
└─ 召回协调
```

## 合规检查清单

- [ ] 年度ISC监控计划是否按时提交？
- [ ] 季度测试结果报告是否按时提交？
- [ ] 是否有即时不合规通知的SOP和响应时效？
- [ ] OTA相关报告是否按Category要求执行？
- [ ] 年度综合ISC报告是否在4月底前提交？
- [ ] 所有ISC数据记录保存期是否≥10年？
- [ ] 记录格式是否机器可读、可检索？
- [ ] 是否有EU法人或代表负责报告提交？
- [ ] 报告是否提供英文版本？
- [ ] 召回相关文档是否保存15年？
- [ ] 客户投诉数据是否纳入ISC分析？

## 常见不合规情况

1. **报告延期**：年度计划提交晚，影响TAA评审节奏。
2. **报告模板不符**：直接使用中国版格式，EU TAA退回要求重写。
3. **即时通知滞后**：单车严重超标72小时后才通知。
4. **VIN与数据脱节**：测试报告未关联VIN，无法追溯。
5. **OTA报告碎片化**：多次Category B更新未合并汇总。
6. **数据保存<10年**：企业数据保留政策与法规冲突。
7. **召回完成率低**：中国车企经销商网络弱，召回完成率<80%被TAA警告。
8. **多车型混合报告**：不同家族合并报告，模糊具体合规状态。
