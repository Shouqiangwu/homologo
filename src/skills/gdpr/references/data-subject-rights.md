# 车辆场景下的数据主体权利

> GDPR Art.12-22 + EDPB Guidelines 01/2020

---

## 权利实现的特殊挑战

车辆场景与传统互联网场景不同，数据主体权利的实现面临独特挑战：多用户共享车辆、无屏幕/键盘的交互限制、驾驶中的安全考虑等。

## 权利清单与车辆场景实现

| 权利 | GDPR条款 | 车辆场景实现方式 | 注意事项 |
|------|---------|----------------|---------|
| **知情权** | Art.13-14 | 分层告知：(1)购车时书面告知 (2)车载屏幕图标+详细说明 (3)APP/网页完整隐私政策 | EDPB推荐使用标准化图标 |
| **访问权** | Art.15 | 通过APP或OEM网站下载个人数据副本 | 需区分不同数据类型的可访问性 |
| **更正权** | Art.16 | 通过APP/经销商更正个人信息 | 主要涉及账户信息，车辆技术数据通常不可更正 |
| **删除权** | Art.17 | 车载"一键删除"按钮（EDPB推荐） | 导航历史、配对手机记录等需可删除 |
| **限制处理权** | Art.18 | 通过设置关闭特定数据收集功能 | 不得影响法律强制功能（eCall、EDR） |
| **数据可携权** | Art.20 | 以机器可读格式导出个人数据 | EDPB建议解释可携带的数据类型 |
| **反对权** | Art.21 | 随时可反对基于合法利益的处理 | 反对远程诊断数据收集等 |
| **自动化决策** | Art.22 | 驾驶行为评分/保险定价需告知逻辑 | UBI保险场景需特别关注 |

## 车辆转售/转让的数据处理

| 场景 | 要求 |
|------|------|
| 车辆出售 | 前车主的所有个人数据必须删除（包括导航历史、蓝牙配对、账户绑定等） |
| 实现方式 | OEM应提供"恢复出厂设置"功能，确保完全删除 |
| 新车主 | 不得访问前车主的任何数据 |
| 云端数据 | 前车主的云端账户必须与车辆解绑，云端数据按保留政策处理 |

## 用户偏好管理

EDPB建议OEM在车辆中实现**用户档案管理系统**（profile management）：
- 不同驾驶员可建立独立档案
- 每个档案有独立的数据处理同意设置
- 切换驾驶员时自动加载对应的隐私偏好

## 合规检查清单

- [ ] 是否提供了分层的隐私告知（书面+车载+APP/网页）？
- [ ] 是否在车载屏幕上提供数据收集状态的可视化指示？
- [ ] 是否提供了"一键删除"功能（导航历史、配对设备等）？
- [ ] 是否支持通过APP/网页行使访问权和数据可携权？
- [ ] 车辆转售时是否有完整的数据清除流程？
- [ ] 是否实现了用户档案管理系统（多驾驶员场景）？
- [ ] 法律强制功能（eCall、EDR）是否在用户行使权利时得到豁免？

## 常见不合规情况

1. **无车端隐私设置界面**：用户只能通过APP或致电客服修改隐私设置，车端无直接操作入口。
2. **转售时数据未清除**：前车主的导航记录、蓝牙设备等在二手车中仍可访问。
3. **删除不彻底**：用户执行"删除"后数据仅在前端隐藏，后端/云端仍保留。
4. **无数据可携功能**：用户无法导出自己的驾驶数据。
