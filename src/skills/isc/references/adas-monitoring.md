# ADAS在用性能监控

> EU 2019/2144（GSR） + EU 2018/858 Art.52 + 相关UNECE法规

---

## ADAS ISC 的独特性

与排放ISC相比，ADAS（AEB、ISA、ELKS、DDAW、ADDW等）的在用监控面临独特挑战：

| 维度 | 排放ISC | ADAS ISC |
|------|---------|---------|
| 测试方法成熟度 | 40+年经验（WLTP/RDE标准化） | 新兴，无标准化ISC测试程序 |
| 性能可量化 | 排放量g/km | 需功能性场景测试 |
| 失效可检测 | OBD系统+OBFCM | 仅当驾驶员察觉或事故时 |
| 数据来源 | OBD接口+年检读取 | OBD+用户反馈+事故数据+远程诊断 |
| 退化预测 | 较成熟 | 初步阶段 |

## 受ADAS ISC约束的系统

### GSR Phase 1（2022-07起新车型，2024-07起所有新车）

| 系统 | UNECE/EU法规 | ISC重点 |
|------|------------|---------|
| AEB（C2C） | R152 | 探测距离、制动启动时机 |
| ISA | EU 2021/1958 | 限速识别准确度 |
| ELKS | R130 | 车道线识别与纠偏 |
| DDAW | EU 2021/1341 | 驾驶员状态监测准确度 |
| EDR | EU 2022/545 | 数据记录完整性 |
| R155 CSMS | R155 | 网络安全事件响应 |
| R156 SUMS | R156 | OTA更新后性能维持 |

### GSR Phase 2（2024-07起）

| 系统 | ISC重点 |
|------|---------|
| AEB扩展（C2P/C2B） | 行人/骑行者探测 |
| ADDW | 分心驾驶监测 |

## 性能退化的主要原因

### 传感器老化

| 传感器 | 退化原因 |
|--------|---------|
| 毫米波雷达 | 雷达罩积灰/划伤、散热不良 |
| 摄像头 | 镜头脏污、内部结露、图像传感器漂移 |
| LiDAR | 光窗污染、内部光学件漂移 |
| 超声波 | 膜片老化、检测范围缩小 |

### 传感器校准失效

| 场景 | 影响 |
|------|------|
| 风挡更换未校准 | AEB摄像头对准偏差 |
| 轮对移位（事故后） | LKA车道线识别偏差 |
| 保险杠更换未对零 | 前雷达角度偏差 |
| 温度漂移 | 长期使用累积偏差 |

### 软件逻辑偏差

| 场景 | 影响 |
|------|------|
| OTA更新引入bug | 新版本性能退化 |
| A/B测试异常 | 部分用户收到实验代码 |
| 与其他ECU通信错误 | 触发条件判断失准 |

## 监控机制

### 1. OEM主动监控（售后数据回传）

若车辆联网且用户同意：

| 数据 | 用途 |
|------|------|
| ADAS激活次数 | 检测异常（激活太多/太少） |
| 误触发事件 | 识别设计问题 |
| 用户关闭行为 | 用户体验反馈 |
| 诊断故障码（DTC） | 硬件故障 |
| 环境传感器数据 | 验证识别准确性 |

**GDPR约束：** 数据收集需用户同意，最小化原则，需匿名化或假名化。

### 2. 客户投诉分析

| 数据源 | 分析要点 |
|--------|---------|
| 经销商报修 | 高频ADAS相关报修 |
| 消费者投诉（CPC网络） | EU消费者投诉系统 |
| 社交媒体/论坛 | 用户讨论（需法律合规方式收集） |
| Insurance claims | 保险理赔事故模式 |

### 3. 事故数据分析

| 数据源 | 分析内容 |
|--------|---------|
| EDR（事件数据记录器） | 事故前5秒ADAS状态 |
| 事故重建 | ADAS应否避免事故 |
| 监管报告 | 发生ADAS相关事故向TAA通报 |

### 4. 周期性实车测试

OEM或独立机构（如Euro NCAP）：

| 测试 | 频次 |
|------|------|
| 新车基准测试 | 型式认证时 |
| 批次抽查 | 每年1-2次 |
| 老化车辆测试 | 针对3-5年车龄样本 |
| 召回评估 | 问题触发时 |

## OTA更新对ADAS ISC的影响

### 影响分类（按R156 SUMS）

| 更新类别 | ISC影响 |
|---------|---------|
| Category A（安全关键，改变ADAS功能） | 需重新型式认证 + 可能触发ISC重新评估 |
| Category B（安全相关，调整参数） | 更新RXSWIN，通知认证机构 |
| Category C（非安全相关） | 仅记录，无需通知 |

### RXSWIN（Regulation X Software Identification Number）

- 每个受R155/R156约束的软件模块有一个RXSWIN
- OTA更新后RXSWIN需更新
- 型式认证文件关联RXSWIN
- 合规软件版本的白名单

## 具体ADAS系统的ISC关注点

### AEB（自动紧急制动）

| 关注点 | 检测方法 |
|--------|---------|
| 误触发率 | 用户反馈+事故数据 |
| 检测距离退化 | 实车测试 |
| 夜间/低能见度性能 | 专项测试 |
| C2P/C2B覆盖 | 量产后真实场景 |

### ISA（智能限速辅助）

| 关注点 | 检测方法 |
|--------|---------|
| 限速标志识别率 | 摄像头性能退化 |
| 地图数据更新 | OTA推送频率 |
| 驾驶员覆盖操作 | 是否易被关闭 |

### ELKS（紧急车道保持）

| 关注点 | 检测方法 |
|--------|---------|
| 车道线识别（褪色/雨雪） | 实车长期测试 |
| 转向干预强度 | EDR记录 |
| 用户关闭率 | 警示体验不佳时 |

### DDAW/ADDW（驾驶员监测）

| 关注点 | 检测方法 |
|--------|---------|
| 误报率（误判清醒为疲劳） | 用户投诉 |
| 漏报率（真正疲劳未识别） | 事故数据 |
| 光照条件适应性 | 黎明/黄昏 |
| 眼镜/口罩场景 | 真实用户反馈 |

## 合规检查清单

- [ ] 是否建立了ADAS在用性能监控框架？
- [ ] 车辆是否有联网能力回传ADAS使用数据？
- [ ] 数据收集是否符合GDPR（用户同意、匿名化）？
- [ ] 是否有客户投诉数据的系统化分析？
- [ ] EDR数据是否在事故调查中使用？
- [ ] 是否做了车龄3-5年的老化ADAS测试？
- [ ] OTA更新是否按R156分类管理？
- [ ] RXSWIN是否维护在认证文件中？
- [ ] 传感器校准建议是否在用户手册中？
- [ ] 售后维修是否包含ADAS校准？

## 常见不合规情况

1. **风挡更换后摄像头未校准**：独立维修店换风挡后未做动态校准，AEB偏差。
2. **传感器清洁不提示**：雷达罩积雪/积灰，AEB失效但无警告。
3. **OTA未分类管理**：所有更新都视为"Category C"，未按R156上报安全相关变化。
4. **用户关闭ADAS未监控**：大比例用户永久关闭ISA/ELKS反映设计问题。
5. **事故数据未回流**：EDR数据存在车内未被OEM分析。
6. **低温/雨雪性能退化**：北欧市场冬季性能大幅退化未被识别。
7. **GDPR合规不到位**：ADAS数据收集未获用户同意即上传。
8. **RXSWIN混乱**：OTA后RXSWIN未及时更新，认证机构抽查时不一致。
