# 中国↔欧盟跨境数据传输

> GDPR Art.44-49 + 中国PIPL跨境传输规则

---

## 核心问题

中国**无EU充分性认定**（adequacy decision）。这意味着从EU向中国传输个人数据需要依赖其他法律机制（GDPR Art.46-49），不能自由传输。

## 双向合规挑战

中国车企在EU运营时面临**双向**跨境数据传输合规要求：

| 方向 | 场景 | 适用法律 |
|------|------|---------|
| **EU→中国** | EU车辆采集的数据传回中国总部（远程诊断、AI训练、质量改进） | **GDPR** Art.44-49 |
| **中国→EU** | 中国开发的软件/数据推送到EU车辆（OTA更新、地图数据） | **PIPL** Art.38-43 |

## EU→中国传输：GDPR合规机制

由于中国无充分性认定，可用的传输机制如下（按推荐优先级排序）：

| 机制 | GDPR条款 | 适用条件 | 难度 |
|------|---------|---------|------|
| **标准合同条款（SCCs）** | Art.46(2)(c) | 与中国数据接收方签署EU标准合同条款 + TIA（传输影响评估） | ⭐⭐ 中等 |
| **约束性公司规则（BCRs）** | Art.47 | 跨国集团内部数据传输的约束性规则，需获DPA批准 | ⭐⭐⭐ 高 |
| **同意（例外情形）** | Art.49(1)(a) | 仅限偶发性传输，不得作为系统性传输的基础 | ⭐ 低但受限 |
| **合同必要（例外情形）** | Art.49(1)(b) | 仅限履行数据主体合同所必要的传输 | ⭐ 低但受限 |

### SCCs + TIA（推荐方案）

| 步骤 | 具体要求 |
|------|---------|
| 1. 签署SCCs | 使用EU 2021/914标准合同条款（Module 1-4，根据控制者/处理者关系选择） |
| 2. 传输影响评估（TIA） | 评估中国法律是否允许数据接收方履行SCCs义务（重点：中国的政府数据访问权力） |
| 3. 补充措施 | 如TIA发现中国法律环境存在风险，需实施补充措施（加密、伪匿名化等） |
| 4. 记录保存 | 保存TIA记录，应DPA要求提供 |

### TIA中的中国法律环境评估要点

| 评估项 | 中国法律现状 |
|--------|------------|
| 政府数据访问 | 中国CSL/DSL/PIPL赋予政府广泛的数据访问权力 |
| 数据本地化要求 | CIIO和重要数据处理者有本地化存储要求 |
| 国家安全例外 | 国家安全审查可覆盖任何数据 |
| **补充措施建议** | 传输前加密（加密密钥留在EU）、伪匿名化、数据最小化 |

## 中国→EU传输：PIPL合规

| PIPL机制 | 适用条件 | 说明 |
|----------|---------|------|
| CAC安全评估 | CIIO或大量个人信息（≥100万人/年）或敏感个人信息（≥1万人/年） | 中国车企如大规模处理EU用户数据可能触发 |
| 标准合同（PIPL SCCs） | 非CIIO且数据量较小 | 2024年3月新规后门槛提高，部分场景豁免 |
| 个人信息保护认证 | 替代方案 | 获认证后可免于其他机制 |
| 豁免情形 | 合同履行必需、跨境电商等 | 2024年3月新规增加了豁免范围 |

## 车联网典型跨境传输场景

| 场景 | 数据流向 | 合规建议 |
|------|---------|---------|
| 远程诊断数据上传中国总部 | EU→中国 | SCCs + TIA + 数据最小化 |
| DMS/ADAS数据用于中国AI训练 | EU→中国 | **高风险** — 建议改为EU本地处理或联邦学习 |
| OTA更新推送 | 中国→EU | PIPL合规 + R156 SUMS |
| 地图/导航数据更新 | 中国→EU | 通常不涉及EU个人数据 |
| 用户投诉/客服数据 | EU→中国 | SCCs + 仅传输必要信息 |
| 全球质量数据分析 | EU→中国 | SCCs + 匿名化/伪匿名化 |

## 合规检查清单

- [ ] 是否识别了所有EU→中国的个人数据传输流？
- [ ] 是否签署了EU SCCs（2021/914版）？
- [ ] 是否完成了TIA（传输影响评估）？
- [ ] TIA是否评估了中国法律环境对SCCs履行的影响？
- [ ] 是否实施了补充措施（加密、伪匿名化等）？
- [ ] 中国→EU的传输是否满足PIPL要求？
- [ ] 同意是否**不**被用作系统性跨境传输的基础？
- [ ] 是否可将部分数据处理本地化到EU以减少跨境传输？

## 常见不合规情况

1. **无SCCs即传输数据**：中国车企的T-BOX直接将数据传回中国服务器，未签署SCCs。
2. **未做TIA**：签署了SCCs但未评估中国法律环境对SCCs履行的影响。
3. **DMS/ADAS数据传回中国用于AI训练**：高风险场景，EDPB推荐车端本地处理。
4. **以"同意"作为系统性传输基础**：GDPR Art.49(1)(a)的同意仅限偶发性传输，不适用于持续性车联网数据传输。
5. **PIPL合规忽视**：仅考虑GDPR，未同步满足PIPL跨境传输要求。
