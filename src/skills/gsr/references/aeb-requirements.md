# AEB 自动紧急制动系统要求

> UNECE R152（M1/N1车辆）

---

## 法规版本演进

| 版本 | 主要内容 | 强制日期（新车型） | 强制日期（所有新车） |
|------|---------|-------------------|---------------------|
| R152.00 | Car-to-Car（C2C）基础要求 | 2022-07-06 | 2024-07-07 |
| R152.01 | 增强行人检测要求（更高速度范围） | 2024-07-07 | 2026-07-07 |
| R152.02 | 新增骑行者检测要求 | 2024-07-07 | 2026-07-07 |

**注意：** 2024年5月1日后首次颁发的型式认证必须基于R152.01或更高版本。R152.00在2024-05-01之前颁发的认证在2026-05-01前仍有效。

## Car-to-Car（C2C）场景 — R152 §5.2.1

### 功能要求

| 要求项 | 具体阈值 | 法规条款 |
|--------|---------|---------|
| 系统激活速度范围 | 10–60 km/h | §5.2.1.2 |
| 碰撞警告触发时机 | 紧急制动开始前≥0.8秒 | §5.2.1.1 |
| 警告模式 | 至少2种（声觉/触觉/视觉中选2） | §5.2.1.1 |
| 紧急制动减速度 | ≥5 m/s² | §5.2.1.3 |
| 驾驶员可中断 | 是（踩油门、转向、打方向灯等正向动作） | §5.3.2 |
| 系统可关闭 | 仅逐个关闭，点火重启后自动恢复 | §5.4.1 |

### 速度减少性能要求 — 静止目标车辆（C2C Stationary）

测试速度：20, 42, 60 km/h（容差+0/-2 km/h）

| 测试车速(km/h) | 最大残余碰撞速度(km/h) — 空载 | 最大残余碰撞速度(km/h) — 满载 |
|---------------|-------------------------------|-------------------------------|
| 20 | 0（完全避免碰撞） | 0 |
| 42 | 0 | 0 |
| 60 | 35 | 35 |

**→ 关键判断标准：** 42 km/h及以下必须完全避免碰撞（零接触）；60 km/h时速度减少≥25 km/h。

### 速度减少性能要求 — 慢速移动目标车辆（C2C Moving）

测试车速 / 目标车速：30/20, 60/20 km/h

| 测试车速/目标车速 | 最大残余碰撞速度(km/h) |
|------------------|----------------------|
| 30/20（相对速度10） | 0 |
| 60/20（相对速度40） | 15 |

## Car-to-Pedestrian（C2P）场景 — R152 §5.2.2

### 功能要求

| 要求项 | 具体阈值 | 法规条款 |
|--------|---------|---------|
| 系统激活速度范围 | 20–60 km/h | §5.2.2.2 |
| 行人横穿速度 | 5 km/h（标准步行速度） | §5.2.2 |
| 紧急制动减速度 | ≥5 m/s² | §5.2.2.3 |
| 碰撞警告 | 至少2种模式，不晚于紧急制动开始 | §5.2.2.1 |

### 速度减少性能要求 — 横穿行人

测试速度：20, 42, 60 km/h

| 测试车速(km/h) | 最大残余碰撞速度(km/h) |
|---------------|----------------------|
| 20 | 0（完全避免碰撞） |
| 42 | 0 |
| 60 | 35 |

**R152.01增强要求（2024-05-01起新认证）：** 行人检测性能要求提高，更高速度范围内要求更大的速度减少。

## Car-to-Bicycle（C2B）场景 — R152.02 §5.2.3

### 功能要求

| 要求项 | 具体阈值 | 法规条款 |
|--------|---------|---------|
| 系统激活速度范围 | 20–60 km/h | §5.2.3.2 |
| 骑行者横穿速度 | 15 km/h | §5.2.3 |
| 紧急制动减速度 | ≥5 m/s² | §5.2.3.3 |

**适用条件限制：** C2B性能要求仅在以下条件下要求达标：
- 无歧义场景（非多辆自行车同时出现）
- 直线行驶，无弯道
- 不在交叉路口转弯

## 误触发控制 — R152 §5.6

| 要求项 | 具体要求 |
|--------|---------|
| 静止旁车道车辆 | 两辆M1轿车停放在两侧，测试车通过中间，不得触发AEB |
| 静止路旁行人 | 行人假人站立在车辆行驶路线旁，不得触发AEB |
| 测试速度范围 | 与对应场景（C2C/C2P/C2B）的测试速度范围相同 |

## 测试程序关键要求 — R152 §6

| 测试条件 | 要求 |
|---------|------|
| 重复次数 | 每个场景测试2次；若1次不通过可增加1次，2/3通过即算通过 |
| 目标假人/车辆 | 必须使用符合规范的GVT（Global Vehicle Target）和行人/骑行者假人 |
| 横向偏移 | 测试车与目标中心线偏移≤0.2m |
| TTC起始距离 | 功能测试起始点：测试车以恒定速度行驶，距目标TTC≥4秒 |
| 环境条件 | 未明确规定夜间测试要求（与US FMVSS 127不同） |
| 路面条件 | 干燥路面，附着系数未明确规定最低值 |

## 故障检测和警告 — R152 §5.5

| 要求项 | 具体要求 | 法规条款 |
|--------|---------|---------|
| 电气故障警告 | 车速>10km/h后≤10秒内激活警告信号，持续至修复 | §5.5.4 |
| 非电气故障 | 传感器遮挡/偏移等，同样需警告 | §5.5.5 |
| 点火重启后 | 如故障仍存在，警告必须重新激活 | §5.5.4 |

## 合规检查清单

- [ ] AEB系统是否覆盖C2C（车对车）场景？
- [ ] AEB系统是否覆盖C2P（车对行人）场景？（R152.01，2024-07-07起新车型强制）
- [ ] AEB系统是否覆盖C2B（车对骑行者）场景？（R152.02，2024-07-07起新车型强制）
- [ ] C2C静止目标：42km/h能否完全避免碰撞？60km/h残余速度≤35km/h？
- [ ] C2P横穿行人(5km/h)：42km/h能否完全避免碰撞？
- [ ] C2B横穿骑行者(15km/h)：相关测试场景是否通过？
- [ ] 碰撞警告是否在紧急制动前≥0.8秒触发（C2C场景）？
- [ ] 警告是否至少采用2种模式（声觉+触觉+视觉中选2）？
- [ ] 紧急制动减速度是否≥5 m/s²？
- [ ] 驾驶员是否可通过正向动作（油门/转向等）中断AEB？
- [ ] AEB关闭是否需逐个操作，点火重启后是否自动恢复？
- [ ] 误触发测试是否通过（旁车道静止车辆/路旁行人不触发）？
- [ ] 电气故障检测和警告功能是否符合要求？
- [ ] 传感器遮挡/偏移是否有故障警告？
- [ ] 是否提供了符合Annex 6的复杂电子系统安全文档？

## 常见不合规情况

1. **仅通过Euro NCAP但未做R152型式认证测试**：Euro NCAP的AEB评分标准与R152法规要求不完全一致（测试场景、通过标准、速度范围均有差异）。Euro NCAP高分不等于R152合规。
2. **C2P行人假人规格不符**：R152使用特定的行人假人规格，与Euro NCAP或中国标准的假人可能不同。
3. **42km/h零接触要求未满足**：部分系统在40-45km/h范围内仅能减速但无法完全避免碰撞。
4. **驾驶员中断逻辑不合规**：中断动作列表未提交给Technical Service，或中断后系统行为不符合要求。
5. **OTA更新影响AEB标定**：AEB软件通过OTA更新后未重新做型式认证验证（涉及R156 SUMS要求）。
