# 车辆数据类型分类和处理要求

> EDPB Guidelines 01/2020 §28-44

---

## 数据分类框架

EDPB将车联网产生的数据分为以下关注等级：

### 高关注数据（需严格保护）

| 数据类型 | 来源 | GDPR分类 | EDPB特别关注 |
|---------|------|---------|-------------|
| **位置/GPS轨迹** | GPS模块、地图服务 | 个人数据 | 可揭示生活习惯、宗教、性取向等 → 可能触发Art.9 |
| **DMS面部图像/模板** | 驾驶员监测摄像头 | **Art.9生物识别数据** | 必须车端本地处理；如用于身份识别，需明确同意 |
| **驾驶行为数据** | CAN总线、加速/制动/转向 | 个人数据 | 可用于保险定价（UBI）、违法推断 |
| **语音数据** | 车载语音助手 | 个人数据 | 语音本身可能构成生物识别数据 |
| **车内摄像头图像** | 座舱摄像头 | 个人数据/生物识别 | 乘客面部、儿童在场等 |

### 中关注数据

| 数据类型 | 来源 | GDPR分类 | 说明 |
|---------|------|---------|------|
| **车外摄像头图像** | ADAS前视/环视摄像头 | 个人数据（含行人面部） | 行人/骑行者面部被采集时构成个人数据 |
| **V2X通信数据** | V2X模块 | 可能含个人数据 | 车辆标识符可能与个人关联 |
| **充电行为数据** | BMS/充电接口 | 个人数据 | 充电时间/地点可推断行为模式 |
| **车辆诊断数据** | OBD/远程诊断 | 可能含个人数据 | 与VIN结合时构成个人数据 |

### 低关注数据（但仍需注意）

| 数据类型 | 来源 | 说明 |
|---------|------|------|
| 车辆技术参数 | ECU | 如不与VIN/个人关联，可能非个人数据 |
| 匿名化统计数据 | 聚合处理后 | 真正匿名化的数据不受GDPR约束 |

## VIN（车辆识别号）的GDPR地位

EDPB和行业（ACEA）对VIN是否构成个人数据有争议：

| 立场 | 论点 |
|------|------|
| **EDPB（偏保守）** | VIN可通过车辆登记数据库与车主关联，构成可识别个人的数据 → 按个人数据对待 |
| **ACEA（行业立场）** | VIN本身不直接识别个人，需额外信息才能关联 → 是伪匿名化数据 |

**→ 合规建议：** 按个人数据对待VIN，避免争议。与VIN关联的所有车辆数据都应视为个人数据。

## 数据处理场景矩阵

| 处理场景 | 典型数据 | 推荐合法性基础 | 处理位置 |
|---------|---------|-------------|---------|
| ADAS功能运行 | 摄像头/雷达实时数据 | 法律义务(Art.6(1)(c)) / 合法利益 | 车端本地 |
| DMS疲劳/分心检测 | 面部图像 | 法律义务(GSR要求) + Art.9明确同意 | **必须**车端本地 |
| GPS导航 | 位置数据 | 合同(Art.6(1)(b)) | 车端+地图服务云端 |
| 远程诊断 | 车辆状态数据+VIN | 同意(Art.6(1)(a)) | 云端 |
| OTA软件更新 | 车辆配置+VIN | 合同/合法利益 | 云端 |
| 驾驶行为分析（UBI保险） | 加速/制动/转向数据 | **同意** | 云端 |
| 事故数据（EDR） | 碰撞前后车辆数据 | 法律义务(Art.6(1)(c)) | 车端本地 |
| 语音助手 | 语音命令/对话 | 同意 | 车端+云端 |
| 车外环视录像 | 环境摄像头视频 | 合法利益（安全）/ 同意 | 车端为主 |

## 合规检查清单

- [ ] 是否建立了车辆数据分类清单（data inventory）？
- [ ] 是否识别了所有Art.9特殊类别数据（DMS、语音、位置推断）？
- [ ] VIN相关数据是否按个人数据处理？
- [ ] ADAS摄像头采集的行人图像是否有数据保护措施（如模糊化、不存储）？
- [ ] 车外摄像头数据是否在车端处理后立即删除原始图像？
- [ ] 是否为每个数据类型明确了处理位置（车端/云端）？
- [ ] 云端处理的数据是否有明确的合法性基础？
